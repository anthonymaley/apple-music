## What Was Done
- Fixed Speaker TUI crash — Swift exclusivity violation where `onToggle` closure captured `items` by reference while `runMultiSelectList` held exclusive `inout` access. Fix: read from separate `devices` array instead of `items`.
- Fixed play cold-start -1728 error — `name of current track` fails immediately after `play` because track hasn't loaded. Added retry loop in `showNowPlaying(waitForPlay:)` that retries up to 10 times (0.3s apart) waiting for playback state to transition from "stopped".
- Added ←→ volume control to speaker TUI — new `onAdjust` callback on `MultiSelectList` component. Speaker TUI passes closure that adjusts volume via AppleScript, returns updated sublabel. Volume changes apply instantly.
- Added playlist browser action sub-menu — after selecting a playlist, shows multi-select menu: "List tracks", "Shuffle play", "Play in order". Can combine actions (e.g. list + shuffle).
- Playlist tracks now write to song cache — `showPlaylistTracks` writes to `last-songs.json` in both API and AppleScript paths, so `music play <index>` works after browsing playlist tracks.
- Refactored all play/skip/back commands to use `showNowPlaying()` — displays full now-playing info (track, artist, album, active speakers with volumes) after every playback action.
- Removed `waitForTrack` constant — all retry logic consolidated in `showNowPlaying(waitForPlay:)`.
- Restructured README into three clear sections: Slash Commands (instant, zero cost), CLI Commands (terminal, scriptable), Natural Language/Skill (AI orchestration).
- Updated header to "for Claude Code & CLI".
- Updated docs (guide.md, SKILL.md) with new features and gotchas.

## Key Decisions
- `showNowPlaying(waitForPlay:)` has two modes: `waitForPlay: true` retries even on "stopped" state (used after play commands), `false` returns "Nothing playing" immediately (used for `music now`).
- `onAdjust` callback returns a new sublabel string rather than modifying items directly — avoids another exclusivity violation.
- Playlist browser uses `MultiSelectList` (not `ListPicker`) for the action menu so users can combine actions (list + shuffle, etc.).
- README restructure: slash commands, CLI, and skill are separate sections with a summary table showing the three layers and their token cost.
