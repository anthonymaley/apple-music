# Session — 2026-04-02

**Machine:** Mac.attlocal.net

## What Was Done
- Fixed Speaker TUI crash — Swift exclusivity violation where `onToggle` closure captured `items` by reference while `runMultiSelectList` held exclusive `inout` access. Fix: read from separate `devices` array instead of `items`.
- Fixed play cold-start -1728 error — `name of current track` fails immediately after `play` because track hasn't loaded. Added retry loop in `showNowPlaying(waitForPlay:)` that retries up to 10 times (0.3s apart) waiting for playback state to transition from "stopped".
- Added ←→ volume control to speaker TUI — new `onAdjust` callback on `MultiSelectList` component. Speaker TUI passes closure that adjusts volume via AppleScript, returns updated sublabel. Volume changes apply instantly.
- Added playlist browser action sub-menu — after selecting a playlist, shows multi-select menu: "List tracks", "Shuffle play", "Play in order". Can combine actions (e.g. list + shuffle).
- Playlist tracks now write to song cache — `showPlaylistTracks` writes to `last-songs.json` in both API and AppleScript paths, so `music play <index>` works after browsing playlist tracks.
- Refactored all play/skip/back commands to use `showNowPlaying()` — displays full now-playing info (track, artist, album, active speakers with volumes) after every playback action.
- Removed `waitForTrack` constant — all retry logic consolidated in `showNowPlaying(waitForPlay:)`.
- Restructured README into three clear sections: Slash Commands (instant, zero cost), CLI Commands (terminal, scriptable), Natural Language/Skill (AI orchestration).
- Updated header to "for Claude Code & CLI".
- Updated docs (guide.md, SKILL.md) with new features and gotchas.

## Key Decisions
- `showNowPlaying(waitForPlay:)` has two modes: `waitForPlay: true` retries even on "stopped" state (used after play commands), `false` returns "Nothing playing" immediately (used for `music now`).
- `onAdjust` callback returns a new sublabel string rather than modifying items directly — avoids another exclusivity violation.
- Playlist browser uses `MultiSelectList` (not `ListPicker`) for the action menu so users can combine actions (list + shuffle, etc.).
- README restructure: slash commands, CLI, and skill are separate sections with a summary table showing the three layers and their token cost.

### Now Playing TUI (`music now`)
- Built interactive now-playing with chafa album art (true color, half-block characters)
- 3-zone layout: album art (left), metadata with progress bar (center), queue (right)
- Progress bar: `0:00 ─────────●──────── 5:56` style with knob indicator
- Output/Mix/Volume block below progress
- Queue shows surrounding tracks from current playlist
- Controls: ↑↓ skip, ←→ seek ±30s, Space pause, s speakers, v volume, r radio, q quit
- Art and queue refresh only on track change (cached)

### Radio Command (`music radio`)
- System Events "Create Station" menu approach doesn't work on macOS 26 (menu item disabled for downloaded tracks)
- Built alternative: searches catalog for artist, creates `__radio__TrackName` temp playlist with 25 tracks, shuffle plays
- Falls back to shuffling local library tracks by artist when no API auth
- `music radio` from terminal drops into now-playing TUI after building playlist

### TUI Design System
- Created `TUILayout.swift` with shared `ScreenFrame`, `renderShell`, `truncText`, `meterBar`
- All screens share coordinates: app label row 2, title row 4, rule row 5, body row 7, footer docked at bottom
- Redesigned speaker picker: inline volume bars for active speakers only, right summary pane
- Redesigned volume mixer: list + levels 2-pane layout
- Redesigned playlist browser: see below

### Playlist Browser (2-screen flow)
- Replaced 3-step flow (list → action menu → track dump) with unified 2-pane browser
- Left pane: playlists. Right pane: scrollable tracks for selected playlist
- Tab switches focus between panes
- Enter on track → transitions to NowPlaying with PlaybackContext (queue = playlist tracks)
- b/Esc in NowPlaying returns to browser with exact position restored
- p = play playlist, s = shuffle playlist (both transition to NowPlaying)
- Preview loads lazily (100ms idle timeout) to keep browser responsive
- Tracks capped at 50 per playlist to avoid AppleScript hanging on large libraries

### Modal Subflows from NowPlaying
- s key opens speaker picker as modal — toggle outputs, adjust volumes, confirm/cancel returns to NowPlaying
- v key opens volume mixer as modal — adjust active speaker levels, returns to NowPlaying
- Both exit raw mode before modal, re-enter after

### Slash Command
- Added `/music:radio` slash command with osascript fallback

## Key Decisions
- chafa `--format symbols` required to avoid iTerm2 inline image protocol (which outputs binary data, not text)
- chafa disabled in playlist browser — conflicts with raw terminal mode. Art only in NowPlaying
- Radio uses catalog search + temp playlist instead of System Events menu click (macOS 26 compatibility)
- Playlist track fetch capped at 50 — "Music" library playlist has thousands of tracks and hangs AppleScript
- Arrow key multi-skip was NOT a parsing bug — it was shuffle mode making it look like multiple skips when the queue was out of order
- Two NowPlaying entry points: `runNowPlayingTUI()` (standalone, `music now`) and `runNowPlayingWithContext()` (from browser, with queue)
